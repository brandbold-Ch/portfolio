Quién soy

Ingeniero de software enfocado en desarrollo backend, con interés en entender los sistemas desde sus capas más fundamentales. Disfruto escarbar en lo primitivo: cómo fluyen los datos, cómo se estructuran los protocolos y cómo pequeñas decisiones impactan la estabilidad y mantenibilidad de un sistema.

Trabajo principalmente en la construcción de APIs REST y servicios backend utilizando Python con FastAPI y Flask, manejando bases de datos relacionales y NoSQL como PostgreSQL, MongoDB y Redis. Me enfoco en validación de datos, diseño claro de endpoints y código predecible.

Aunque Python es mi base actual, mantengo una mentalidad abierta y adaptable: me interesa aprender y trabajar con otros lenguajes y tecnologías backend, como Java y distintos ecosistemas, según las necesidades del proyecto.

Curiosidad Técnica

Más allá del desarrollo de APIs y aplicaciones de negocio, me interesa comprender cómo funcionan internamente los lenguajes, frameworks y herramientas que utilizo diariamente.

Dedico parte de mi tiempo a explorar conceptos avanzados de Python y Java, estudiando mecanismos como reflexión, metaprogramación, mapeo objeto-relacional, inyección de dependencias, sistemas de tipos y arquitectura de frameworks, Unit of Work, Identity Map, AOP, seguimiento de cambios y construcción dinámica de consultas.

Como ejercicio de aprendizaje suelo replicar componentes que normalmente utilizamos como cajas negras. Entre estos experimentos se encuentran pequeños ORMs y mappers para Python y Java, implementaciones inspiradas en patrones de frameworks modernos y proyectos personales orientados a entender decisiones de diseño utilizadas en herramientas reales.

Actualmente desarrollo NestPy, una propuesta inspirada en NestJS para los ecosistemas de FastAPI y Flask, explorando conceptos como módulos, contenedores de dependencias, decoradores y organización estructurada de aplicaciones backend.

Considero estos proyectos una forma de profundizar en la ingeniería de software, entendiendo no solo cómo usar una herramienta, sino también cómo está construida.

Mini ORM para Java

Proyecto experimental desarrollado para comprender cómo funcionan internamente herramientas como Hibernate y JPA. Implementé un sistema de persistencia basado en reflexión y anotaciones personalizadas capaz de generar esquemas de base de datos y consultas SQL a partir de clases Java.

El proyecto incluye procesamiento de anotaciones como @Table, @Column, @Id y @GeneratedValue, construcción de metadata mediante reflexión, generación automática de sentencias CREATE TABLE e INSERT, mapeo de tipos Java a SQL y ejecución dinámica mediante JDBC.

Como parte del diseño implementé un EntityManager propio, un sistema de migraciones automáticas, un generador de consultas SQL y una capa de configuración desacoplada para distintos motores de base de datos.

Tecnologías: Java, Reflection API, JDBC, PostgreSQL, Annotations, Query Generation

NestPy

Framework experimental inspirado en NestJS y desarrollado sobre FastAPI y Flask con el objetivo de comprender cómo funcionan internamente los sistemas modernos de inyección de dependencias, descubrimiento de componentes y registro automático de rutas.

Implementé decoradores para módulos, controladores, servicios y endpoints, junto con un contenedor de aplicación encargado de registrar dependencias, almacenar metadatos y construir la configuración de la aplicación mediante introspección y reflexión.

El framework incorpora escaneo dinámico de clases, análisis de firmas de funciones, construcción automática de rutas, adaptación hacia FastAPI y un sistema de metadatos inspirado en la filosofía de NestJS.

Como parte de la arquitectura desarrollé componentes propios para reflexión, registro de controladores, manejo de módulos, resolución de dependencias y generación dinámica de handlers a partir de información obtenida mediante inspect.

Tecnologías: Python, FastAPI, Flask, Reflection, Dependency Injection, Decorators, Introspection, Framework Design

DBF Mapper

Uno de mis primeros proyectos de exploración técnica en Python, desarrollado durante mi proceso de aprendizaje de reflexión e introspección. El objetivo fue transformar archivos DBF heredados en objetos Python utilizando un modelo similar a un ORM ligero.

Implementé una capa de acceso a datos capaz de leer estructuras DBF, construir modelos dinámicamente y mapear registros a objetos Python utilizando genéricos, anotaciones de tipo y metadatos definidos mediante Annotated.

El proyecto incluye resolución básica de relaciones entre modelos, búsqueda dinámica por criterios, representación automática de entidades y un sistema de lectura desacoplado para interactuar con bases de datos DBF utilizadas en sistemas heredados.

Aunque fue desarrollado antes de profundizar en conceptos como metaclases, decorators avanzados y manipulación de runtime, representó uno de mis primeros acercamientos al diseño de librerías y herramientas de infraestructura en Python.

Tecnologías: Python, Reflection, Generics, Typing.Annotated, DBF, Introspection

Net Keyboard

Proyecto inspirado en herramientas como Synergy, cuyo objetivo es controlar múltiples equipos utilizando un único teclado y mouse, permitiendo compartir eventos de entrada entre diferentes máquinas sin hardware adicional.

Implementé un sistema de captura de eventos de teclado a bajo nivel mediante un listener desarrollado en C, encargado de leer los eventos directamente desde el sistema operativo y transmitirlos hacia una capa superior escrita en Python.

La comunicación entre procesos fue diseñada utilizando IPC mediante sockets Unix, creando un canal de comunicación binario donde los eventos del teclado son serializados como estructuras compactas conteniendo código de tecla, estado y timestamp.

La arquitectura separa la captura del hardware y el procesamiento de eventos, permitiendo desacoplar el componente nativo del backend Python mediante un sistema basado en listeners, callbacks y distribución de eventos.

Este proyecto fue una exploración de programación de sistemas, integración entre C y Python, comunicación entre procesos y manejo de eventos en tiempo real.

Tecnologías: C, Python, IPC, Unix Sockets, Binary Protocol, Event Streaming, Linux
